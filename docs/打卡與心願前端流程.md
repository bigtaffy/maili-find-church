# 打卡與心願前端流程

## 目的

這份文件描述第一階段「無會員、本機版」的前端使用流程。

目標是讓後續實作時，能先對齊：

- 哪些頁面會出現功能入口
- 使用者每一步會看到什麼
- 哪些狀態要提示
- 哪些互動值得保留儀式感

## 第一階段前提

- 沒有會員
- 沒有雲端同步
- 朝聖印章與心願卡保存在 localStorage
- 功能主入口放在教堂詳情頁

## 主要畫面

第一階段先不新增獨立大模組頁，先掛在既有頁面：

1. 教堂詳情頁
2. 收藏頁
3. 設定頁或功能提示區

## 教堂詳情頁流程

### 區塊順序建議

在教堂詳情頁中，於現有內容中新增一個「朝聖印章與心願卡」區塊。

建議順序：

1. 教堂基本資訊
2. 彌撒時間
3. 地圖與導航
4. 朝聖印章
5. 心願卡
6. 錯誤回報

### 狀態一：尚未蓋章，且不在可蓋章範圍

顯示內容：

- 區塊標題：`朝聖印章`
- 一段說明：
  - 到達教堂附近後，可留下這間教堂的朝聖印章
- 按鈕：
  - `尚未到達教堂附近`
- 按鈕為 disabled 狀態

可補充一行小字：

- 需位於教堂附近約 100 公尺內

### 狀態二：尚未蓋章，但已在可蓋章範圍

顯示內容：

- 區塊標題：`朝聖印章`
- 說明：
  - 你已到達教堂附近，可以留下這次到訪記錄
- 主按鈕：
  - `留下朝聖印章`

這裡不建議自動蓋章，應保留手動按下的儀式感。

### 狀態三：蓋章成功

按下後流程：

1. 按鈕進入 loading
2. 顯示蓋章動畫
3. 畫面切換成已完成狀態

顯示內容：

- 狀態標記：`已留下朝聖印章`
- 日期：`2026/03/27`
- 裝飾性印章視覺
- 小文案：
  - 這是你在這間教堂留下的朝聖記錄

按鈕可改成次要樣式：

- `查看我的印章`

### 狀態四：已經蓋過章

再次進入詳情頁時：

- 不再顯示蓋章 CTA
- 直接顯示已蓋章狀態
- 保留日期

可補一個小連結：

- `查看印章收藏`

## 心願卡流程

### 區塊位置

放在朝聖印章區塊下方。

區塊標題：

- `心願卡`

說明文字：

- 每間教堂只有第一次到訪並完成蓋章後，才會解鎖 3 張心願卡
- 心願卡目前只保存在此裝置

### 解鎖條件

- 必須先完成該教堂的第一次朝聖印章
- 未蓋章前，不顯示可編輯的 3 張卡片

### 狀態一：尚未解鎖

顯示內容：

- 區塊標題：`心願卡`
- 說明：
  - 第一次留下朝聖印章後，會解鎖這間教堂的 3 張心願卡
- 視覺狀態：
  - 3 張鎖定卡片或一張鎖定說明卡

不建議：

- 直接顯示可點擊空白卡

### 狀態二：已解鎖

完成首次蓋章後，切換為可使用狀態。

### 卡片結構

每間教堂固定顯示三張卡：

1. 為自己
2. 為家人
3. 為世界

### 狀態三：空白卡

顯示內容：

- 類別名稱
- placeholder
  - `寫下這次想放在這裡的心願`
- 按鈕：
  - `新增心願`

### 狀態四：編輯中

點擊後展開編輯區：

- 多行輸入框
- 狀態選單
  - 靜待中
  - 已實現
  - 放下了
- 操作按鈕
  - `儲存`
  - `取消`

### 狀態五：已儲存

顯示內容：

- 類別
- 心願文字
- 狀態標記
- 最後更新時間

操作：

- `編輯`
- `清除`

## 收藏頁流程

### 第一階段目標

收藏頁先新增一個分頁或區塊：

- `我的印章`

先不把心願卡放進收藏頁。

### 我的印章列表

每張卡顯示：

- 教堂名稱
- 蓋章日期
- 教堂圖片或簡化章面

排序預設：

- 最新蓋章在前

### 空狀態

若還沒有任何印章：

- 顯示空狀態圖或簡單插圖
- 文案：
  - 走進一間教堂後，就能留下第一枚朝聖印章

## 地圖頁流程

第一階段地圖頁只做輕量回饋，不做複雜互動。

### 建議行為

- 已蓋章的教堂 marker 可帶一點微弱識別
- 不要加大面積強提示
- 不要在地圖頁直接做蓋章動作

原因：

- 蓋章應保留在詳情頁完成
- 地圖頁適合探索，不適合儀式型操作

## 引導流程

### 第一次進入教堂詳情頁

若使用者從未看過功能說明，可顯示一次輕量提示：

- 朝聖印章會記錄你到訪過的教堂
- 第一次完成蓋章後，會解鎖這間教堂的心願卡
- 心願卡會保存在此裝置

### 顯示時機

- 第一次看到朝聖印章區塊時
- 或第一次點進心願卡區塊時

### 不建議

- 全螢幕教學遮罩
- 多頁 onboarding

第一階段應保持輕量。

## 例外狀態

### GPS 權限未開

朝聖印章區塊顯示：

- `尚未開啟定位，暫時無法確認是否到達教堂附近`

按鈕：

- `重新取得定位`

### GPS 精度不足

顯示：

- `目前定位精度不足，請靠近教堂或稍後再試`

### localStorage 無法使用

顯示：

- `目前無法在此裝置保存朝聖印章與心願卡`

這種情況雖少，但要有 fallback。

## 動畫建議

### 蓋章動畫

第一階段可先做簡化版：

1. 印章縮放進場
2. 透明度由低到高
3. 蓋章日期淡入

### 心願卡

- 不需要花太多動畫
- 以卡片展開收合與儲存成功回饋為主

## 第一階段 UI 決策

- 朝聖印章主入口在教堂詳情頁
- 心願卡主入口也在教堂詳情頁
- 收藏頁只先收印章
- 地圖頁只做已蓋章微弱識別
- 不做公開牆、不做排行榜、不做社交互動
