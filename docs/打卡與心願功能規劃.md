# 打卡與心願功能規劃

## 功能定位

這個功能不走一般景點式「打卡」，而是做成比較有儀式感、收藏感與回訪動機的體驗。

- 打卡主軸：`朝聖印章`
- 願望主軸：`心願卡`

使用者到教堂後，可以手動蓋章留下到訪記錄；每間教堂則可以保存 3 張心願卡，作為和這間教堂之間的個人紀錄。

## 產品目標

- 讓「到過教堂」有值得保留的紀念感
- 讓教堂不只是地圖點位，而是可累積回憶的地方
- 讓使用者有再次打開 App、再次回訪同一間教堂的理由
- 避免做成社群牆、排行榜或手遊式連續簽到

## 核心概念

### 朝聖印章

- 使用者到達教堂附近後，可以手動蓋章
- 蓋章成功後，會留下該教堂的印章、日期與到訪記錄
- 每間教堂對應一枚專屬印章
- 印章是收藏，不是單純「已到此一遊」

### 心願卡

- 每間教堂只在第一次到訪並完成蓋章後，解鎖 3 張心願卡
- 心願卡是私密內容，預設只有自己看得到
- 心願卡不是留言牆，不做公開串流
- 下一次回到同一間教堂時，可以再回看或更新心願
- 第一版先保存在本機裝置

## 命名建議

### 對外名稱

- 朝聖印章
- 心願卡

### 避免使用

- 打卡
- 願望牆
- 排行榜

原因是這些詞會讓產品氣質偏向一般社群或景點 App，和目前產品方向不太一致。

## MVP 功能範圍

### 1. 教堂蓋章

- 使用者進入教堂指定範圍後，詳情頁顯示 `留下朝聖印章`
- 蓋章必須手動觸發，不自動完成
- 蓋章成功後顯示動畫與日期
- 同一裝置同一間教堂只保留一筆有效印章

### 2. 每間教堂 3 張心願卡

- 一間教堂固定 3 個欄位
- 只有第一次到訪並成功留下朝聖印章後才會解鎖
- 不採用無上限新增
- 每張卡可編輯、更新狀態、刪除內容

### 3. 心願卡欄位

- 類別
  - 為自己
  - 為家人
  - 為世界
- 內容
- 狀態
  - 靜待中
  - 已實現
  - 放下了

### 4. 收藏頁擴充

- 新增 `我的印章`
- 顯示已收集教堂數量
- 顯示最近蓋章紀錄
- 第一版資料只保存在目前裝置

## 互動設計方向

### 蓋章流程

1. 使用者進入教堂範圍
2. 詳情頁出現可用狀態
3. 使用者按下 `留下朝聖印章`
4. 顯示蓋章動畫
5. 寫入印章記錄

### 心願卡流程

1. 使用者進入教堂詳情頁
2. 第一次完成朝聖印章後，解鎖該教堂的 3 張心願卡
3. 使用者看到該教堂的 3 張心願卡槽位
4. 點選空白卡新增內容
5. 後續可更新內容或改變狀態

## 趣味性設計

### 印章

- 每間教堂有不同章面
- 初期可先用教堂名稱縮寫、顏色、簡化圖騰組成
- 後續可升級成教堂輪廓或主保聖人元素

### 蓋章動畫

- 印章下壓
- 墨色擴散
- 出現日期
- 顯示「第 N 間已收集教堂」

### 回訪感

- 再次進入同一教堂時提示：
  - 你曾在這裡留下朝聖印章
  - 你在這裡還有心願卡

## 不建議先做的方向

- 公開願望牆
- 願望按讚或留言
- 教堂收集排行榜
- 連續打卡獎勵
- 強制任務導向的手遊式成就

這些方向雖然會提升互動量，但會破壞目前產品偏安靜、溫和的氣質。

## 前端畫面規劃

### 教堂詳情頁

- 顯示 `留下朝聖印章` 按鈕
- 第一次完成蓋章後顯示 3 張心願卡
- 顯示是否已蓋章
- 顯示最近一次蓋章日期

### 收藏頁

- 新增 `我的印章`
- 可依城市、教區、時間排序

### 地圖頁

- 已蓋章教堂 marker 可帶有微弱識別狀態
- 不建議過度高亮，避免地圖太花

## 資料結構建議

### stamps

- `parish_id`
- `stamped_at`
- `lat`
- `lng`
- `verification_method`

### wishes

- `parish_id`
- `slot`
- `category`
- `content`
- `status`
- `created_at`
- `updated_at`

### 建議限制

- `parish_id` 在 `stamps` 中唯一
- `parish_id + slot` 在 `wishes` 中唯一
- `slot` 固定為 1 到 3

## 驗證與判定邏輯

### 蓋章條件

- 使用者需位於教堂一定距離內
- 初期建議距離：
  - 80m 到 150m 之間擇一
- 實作時可依 GPS 誤差與都市環境再調整

### 驗證方式

- 優先使用裝置 GPS
- 記錄蓋章當下座標
- 若精度過差，可提示稍後再試

## 第一階段資料保存方式

- 目前沒有會員機制
- 第一階段以 localStorage 保存朝聖印章與心願卡
- 不做雲端同步
- 不保證跨裝置保留

## 階段規劃

### 第一階段

- 手動蓋章
- 3 張心願卡
- 私密資料
- 收藏頁顯示我的印章
- localStorage 單機保存

### 第二階段

- 會員機制
- 雲端同步
- 每間教堂專屬印章樣式
- 城市 / 教區蒐集進度
- 回訪提醒

### 第三階段

- 主題巡禮路線
- 特殊節日限定章
- 完整朝聖護照頁

## 實作時的原則

- 儀式感大於遊戲感
- 收藏感大於任務感
- 私密性大於公開互動
- 回訪價值大於一次性打卡
