# 打卡與心願 Wireframe 與文案草稿

## 目的

這份文件把第一階段「朝聖印章 / 心願卡」功能，整理成低保真 wireframe 描述與對應文案。

用途：

- 對齊畫面資訊層級
- 對齊按鈕與提示文案
- 作為之後正式 UI 設計與前端實作的底稿

## 前提

- 無會員
- localStorage 保存
- 只有第一次到訪並完成蓋章後，才解鎖 3 張心願卡

## 畫面一：教堂詳情頁未到達狀態

## 畫面十：例外狀態文案

### GPS 未開

- `尚未開啟定位，暫時無法確認是否到達教堂附近`
- `重新取得定位`

### GPS 精度不足

- `目前定位精度不足，請靠近教堂或稍後再試`

### localStorage 不可用

- `目前無法在此裝置保存朝聖印章與心願卡`

## UI 原則

- 朝聖印章是主儀式，視覺上應比心願卡更突出
- 心願卡較安靜，像私人札記，不要太像任務清單
- 已解鎖與未解鎖狀態要非常清楚
- 第一版先追求乾淨與可理解，不先追求花哨
